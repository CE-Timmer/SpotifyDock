# SpotifyDock

[![Version](https://img.shields.io/badge/version-1.1.0-4c8bf5)](#)
[![Platform](https://img.shields.io/badge/platform-Windows-0078d4)](#)
[![License](https://img.shields.io/badge/license-MIT-2ea043)](./LICENSE)

SpotifyDock is a Windows-first desktop lyrics overlay for Spotify built with Tauri, React, and TypeScript.

It sits at the top of your screen as an always-on-top overlay, follows the current track, renders synced lyrics with smooth transitions, and can use either:

- **DockBridge** mode (powered by original Spicy Lyrics + `dockbridge.js`) for rich synced lyrics
- **LRCLIB + local cache** as a backup mode

## Features

- Always-on-top transparent overlay window
- Frameless top-screen dock with rounded acrylic-style presentation
- Hover-to-hide overlay behavior
- Tray icon with quick actions
- Autostart support
- Dedicated control/settings window
- Optional Spotify auth popup for backup playback mode
- DockBridge source mode for best synced lyrics support
- Backup lyrics mode using LRCLIB + local cache
- Left / center / right lyric alignment
- Secondary sub-lines and parentheses styling
- Word timing support when provided by the bridge
- Album cover and track metadata in the left panel
- Playback progress bar and timer
- Windows media helper or Spotify Web API helper for playback assistance

## How It Works

SpotifyDock supports two lyrics pipelines:

### 1. DockBridge mode

Recommended if you already use Spicetify and want the richest synced lyrics behavior.

Flow:

- `dockbridge.js` provides track + lyric stream directly to SpotifyDock
- original `spicy-lyrics.js` is optional (only needed if you also want its UI in Spotify)
- `dockbridge.js` sends normalized track and lyric payloads to SpotifyDock
- SpotifyDock renders the overlay from that stream

This mode does **not** require Spotify Web API login for lyrics playback.

### 2. Backup mode

Recommended if you do not want to rely on the Spicetify bridge.

Flow:

- SpotifyDock authenticates against the Spotify Web API
- current playback is read from Spotify
- lyrics are fetched from LRCLIB
- results are cached locally and can be overridden with custom JSON files

## Setup

### 1. Install dependencies

```powershell
npm install
```

### 2. Configure environment

Create `.env` in the repo root:

```env
VITE_SPOTIFY_CLIENT_ID=your_spotify_client_id
VITE_SPOTIFY_REDIRECT_URI=http://localhost:5173
```

These values are used for backup mode and Spotify Web API helper flows.

### 3. Create Spotify Developer app

If you want backup mode or the Spotify Web API helper:

1. Go to the Spotify Developer Dashboard
2. Create an app
3. Add your redirect URI
4. Copy the client ID into `.env`

Important:

- the redirect URI origin must match the app origin during development
- if you are running Vite dev, `http://localhost:5173` is the expected origin unless you changed it

### 4. Run in development

```powershell
npm run tauri dev
```

### 5. Build production app

```powershell
npm run build
npm run tauri build
```

## Using DockBridge Mode

SpotifyDock works with `dockbridge.js` alone. Original Spicy Lyrics is optional.

### Recommended extension setup

Required:

- `dockbridge.js`

Optional add-on:

- original `spicy-lyrics.js` if you also want the original Spicy Lyrics UI

### Spicetify extension folder

Typical location:

```text
%APPDATA%\spicetify\Extensions
```

Minimum expected file:

- `dockbridge.js`

Optional additional file:

- `spicy-lyrics.js`

Then run:

```powershell
spicetify apply
```

Restart Spotify after applying.

## Using Backup Mode

In the control window:

- set `Lyrics Source` to `Backup (LRCLIB + Cache)`
- use the Spotify connection popup if needed

Backup mode uses:

- Spotify Web API for current playback
- LRCLIB for lyrics
- local JSON cache / overrides for edits and alignment metadata

## Local Lyrics Cache and Overrides

SpotifyDock stores cached and custom lyric files in the local app data folder.

Each file follows this shape:

```json
{
  "trackId": "spotify_track_id",
  "title": "Song Name",
  "artists": ["Artist A", "Artist B"],
  "durationMs": 180000,
  "source": "override",
  "lyrics": [
    {
      "time": 12.4,
      "text": "Where did you go?",
      "side": "left",
      "singer": "Artist A"
    },
    {
      "time": 15.1,
      "text": "I was right here.",
      "side": "right",
      "singer": "Artist B"
    }
  ]
}
```

Supported line fields:

- `time`
- `duration`
- `text`
- `secondaryText`
- `side`
- `singer`
- `words`

## Notes

- DockBridge mode is the preferred setup for rich synced lyrics behavior
- Backup mode exists for resilience and simpler standalone playback support
- Original Spicy Lyrics is optional and can run separately beside DockBridge
- `dockbridge.js` should be treated as a sidecar extension, not a replacement renderer

## Troubleshooting

### Overlay opens but no lyrics appear

Check:

- `Lyrics Source` is set to `DockBridge`
- `dockbridge.js` is installed
- `spicetify apply` has been run
- Spotify has been restarted

### Backup mode says Spotify is not connected

Check:

- `.env` contains `VITE_SPOTIFY_CLIENT_ID`
- `.env` contains `VITE_SPOTIFY_REDIRECT_URI`
- redirect URI in Spotify Developer Dashboard matches your app origin

### Overlay is hidden

Use one of these:

- tray icon -> `Show Overlay`
- control window -> `Show Overlay`
